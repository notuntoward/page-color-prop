# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public issue for security vulnerabilities.**

### How to Report

- **Email**: Send details to the maintainer at the email address listed on the [GitHub profile](https://github.com/notuntoward)
- **GitHub**: Use [GitHub's private vulnerability reporting](https://github.com/notuntoward/page-color-prop/security/advisories/new)

### What to Include

- Description of the vulnerability
- Steps to reproduce or a proof of concept
- Expected vs actual behavior
- Any suggested fixes (if applicable)

### Response Timeline

- Initial acknowledgment: within 7 days
- Assessment and triage: within 14 days
- Fix timeline: depends on severity and complexity

## Disclosure Policy

Please allow reasonable time for the vulnerability to be addressed before any public disclosure. Coordinated disclosure helps protect users while a fix is being developed.
